Now, for the long awaited (*drum roll and sound of trumpets*) DeadSwifFPS Tutorial!
Ok, here's what you gotta do:

First, make a button for the base of your monster. Draw only the non-moving parts. Do all your drawing in the Up State. When you get done, copy the all the shapes you just made to the Over State layer and the Hit State layer. Change the name of this button to MonsterBase, or, if you plan to make multiple types of monsters, <MonsterType>Base. I named my monster YellowMeanie, so the button name was YellowMeanieBase.

Now, create a new sprite called Monster (or <MonsterType>: mine is YellowMeanie) and add an instance of the button to this area. Align the button to the center of the drawing area using the appropriate item on the Tool menu ([Tool] > [Align] > [Align to canvas center]). Add the following ActionScript to the onPress() (yes, onPress, not onRelease!) of the button:
Code:
_root.monsterKillCount++;
gotoAndPlay("dead");

Go to and play "dead"?!? Have I lost my mind?!? Ummm... nope, have you? Lol, just joking ! I haven't seen this covered in any of the other tutorials, but there is a way to name frames so you dont have to refer to them by number. All you have to do is move to the frame you want to name, and click the area that says "<Frame Label>" and type it in. Then you can refer to the frame even if you decide to move it further along in the movie later without having to change any of the ActionScript.

Ok, well now you need to make a frame called dead. Go to frame 2 and name it the way I just told you. On your clipboard (the temporary holder for whatever you copy and paste) you should still have the monster that you pasted into the HitState. If not, go back to the button and copy it again. Now paste it into the "dead" frame. We need to make sure it is aligned to the monster in the other frame... but how? Let's use a little trick: Group. Select all the shapes of the monster, and hit [ctrl]+[g] on your keyboard (or click [Group] on the Tool menu). Now you can safely align to canvas center without messing anything up. Before you can edit it you have to Un-Group (press [ctrl]+[u] or use the Tool menu). Edit it as you see fit to make it look dead. You may want to add some blood or something.

Now go back to Scene 1. Add an instance of your monster. Preview. Click on it to make it die. Your monster looks dumb. Yes, I said it looks really stupid. Now before you reply to this topic getting all mad, let me explain: it needs moving parts, and it needs to disappear when it dies. Ok, exit the preview and delete that instance (select the monster and hit [Delete] on your keyboard) because it is just that bad. Ok, let's edit this thing!

Ok, go back to your monster sprite. Go to frame 2, select all and cut. Remove the frame label. Go to frame... I don't know, 50 I guess, and paste your dead monster. Make sure you label this frame "dead". Now insert a transform motion on the entire dead monster (you probably have to select it all first). Drag the little blue dot on the right side of the timeline from whatever frame it is on to about 6 frames from the one labeled "dead". Go to that frame, select the dead monster, and click the little drop down box next to all color palette that says "None". Change it to "Alpha" and move the little slider that appears all the way to 100%. Add some ActionScript to onFrame() to make the clip "stop();" at this point. Now go back to the beginning of this sprite and add a transform motion to your monster button. Add another layer. Draw any animations or moving parts to this area. In my case, I created a double tounge (yes, that is what it was supposed to be... I know, it looks awful) that moves around the outside of the monsters's mouth using path motion. You may want to make moving arms or tenticles or something. Just remember that if the user clicks on it, it will not kill the monster because it is not part of the button which you added the ActionScript to. If you want the user to be able to kill it by clicking on the moving parts, you will have to make them a button also and add the code I gave you for the other button. To the last frame of the monster moving, add to onFrame:
Code:
gotoAndPlay(1);


When you are done, go back to Scene 1, add an instance of you monster, and Preview your movie. Now your monster looks much better, good job ! Ok, exit the preview and delete that instance.

Change the name of this layer to "Background", and the name of this frame to "menu". You probably want to change the background color of your movie at this point. Make this look like a game menu with the name of the game, maybe with an instance of your monster, a credits button, instuctions button, and definately a play button. Add this ActionScript to onFrame:
Code:
var monsterKillCount=0;
stop();

When the user clicks (onRelease) on the play button, it should execute the following code:
Code:
_root.monsterKillCount=0; // in case we placed any instances of the monster on this frame and the user has killed them
gotoAndPlay("start");

Skip a few frames (enough so we can add credits, instuctions or whatever later if we wish. Skip 5 frames.). So lets go to frame 6. Label it "start" and draw a background on it. Select all objects in the background and add transform motion: this allows it to remain for the rest of the level. Drag the right-most blue dot of the slider for the background to however long you think your level will be.

OK, now for the monsters! Add a new layer called "Monster 1" and add a monster to it. We are going to want this monster to move around, so lets draw where we will want it to move using the freehand tool (you may want to set the smoothness to 0) or the curve tool. Now select both your monster and the path and insert path motion. Preview. Looks kinda weird, huh? Especialy that one part where it goes upsidedown. Well, we will leave this one jsut as it is for now. Exit the preview.

Add a new layer called "Stuff 1" and add whatever you want to be in front of this monster. Make sure you draw this on the first frame of this level, and add transform motion to make it stick until the end.

Add a new layer for the next monster and rename it "Monster 2". Go to about the middle of the level and add another monster. This time, let's use transform motion to move it around (select the monster and insert transform motion). On the first frame that this monster is in, place it all the way on one side of the drawing area. Now click to the farthest frame that the monster is in, and move it all the way to the other side of the drawing area. Preview. That... is really boring. And he moves really slow and is too easy to kill. This stinks. Ok, exit the preview. We are going to make the monster hop around now. Double click on the blue bar between the 2 blue dots for this monster. Whow! Another blue dot appeared! You can now drag the monster to another position inside that frame and resize or even rotate the monster if we want. Repeat this step to make the moster hop from one side of the screen to the other. Preview. Wow, that is pretty cool... But we need more monsters!! Exit the preview.

Repeat the last three paragraphs creating a new layer for each monster and layers containing stuff for monsters to hide behind whenever we feel like it. Add path motions and transform motions whenever you create a new instance of a monster to make it move around. This is starting to look pretty cool. Ok, now go to the end, the last frame of this level and add the following ActionScript to onFrame:
Code:
if(_root.monsterKillCount==2){ //2 is the number of monsters in this level
   _root.monsterKillCount=0;
   gotoAndPlay("win");
} else {
   gotoAndPlay("lose");
}

and change 2 to the number of monsters you have.

Go to the next frame and label it "win". Decorate it to let the user know they have won and include a button to go back to the menu frame or play again (gotoAndStop("menu"); or gotoAndPlay("start"). Add the code "stop();" to this frame's onFrame. Go to the next frame and do the same thing, but for "lose".

That's it, you're done!

I think I included all the steps, but if something doesn't appear to be working, let me know so I can revise the tutorial. If you want to click the quote button to refer to a specific part of what I said, remove everything else that I said because this is really really long, and we want to know exactly what you are talking about. I will post the .lmv before the day is out (Eastern Daylight Time Zone). If you make a game using this tutorial, don't forget to put me in the credits under "Idea from:" or something like that. I would appreciate it alot !

DeadSwif FPS tutorial

Make a button for the base of your monster
Draw (in the Up state) the non-moving parts for your monster
Duplicate the Up state into the Over and Hit states
Name your button MonsterBase, or <MonsterType>Base. I named mine
YellowMeanieBase (MonsterType=YellowMeanie)

Create a sprite called Monster (or <MonsterType>)
Add an instance of your MonsterBase button, and align it to the center ([Tool] > [Align]
> [Align to canvas center])
in the onPress actionscript area, put:
Quote:
_root.monsterKillCount++;
gotoAndPlay("dead");

move to frame 2 and draw your dead monster. In the area (upper left) that says "<Frame
Label>", type "dead".
now when the button says 'gotoAndPlay("dead")' the sprite will move to that frame.
One way to make your dead picture is to copy your sprite from the hit state of
your "MonsterBase" button, paste it into your "dead" frame, and doctor it.
go back to Scene 1
add an instance of your monster
Preview. Click on it to make it die. Pretty Dumb, huh?
It needs moving parts, and it needs to disappear when it dies.
exit the preview and delete that instance

Ok, let's edit this thing!

Go back to your monster sprite.
Go to frame 2, select all and cut.
Delete the frame label.
Go to some frame (i chose 50), and paste your dead monster.
Label this frame "dead".
Insert a transform motion on the entire dead monster
Extend the transform motion to end about 6 frames ahead of the "dead" frame. (drag the blue dot)
Go to the frame at the end of your motion
Select the dead monster, and click the little drop down box next to all color palette that says "None".
Change it to "Alpha" and move the little slider that appears all the way to 100%.
In the onFrame() actionscript box (of the frame) put "stop();"
Go to the beggining of the sprite and add a transform motion to your monster button.
Add a layer.
Draw any animations or moving parts to this area.
(In my case, I created a double tounge that moves around the outside of the monsters's mouth using path motion.)
You may want to make moving arms or tenticles or something.

If the user clicks these parts, it will not kill the monster because it is not part of the button which you added the ActionScript to. If you want the user to be able to kill it by clicking on the moving parts, you will have to make them a button also and add the code I gave you for the other button.

To the last frame of the "alive" part of the moster, insert into the onFrame actionscript area:
Code:
gotoAndPlay(1);

Go back to Scene 1, add an instance of you monster, and preview your movie.
Now your monster looks much better, good job
Exit the preview and delete that instance.

Menu

Change the name of this layer to "Background", and the name of this frame to "menu".
Change the background color of your movie (if you want)
Make a couple of buttons (play, instructions, credits, etc.) and put the in a menu-looking order.
Add this ActionScript to onFrame:
Code:
var monsterKillCount=0;
stop();

Put this in the onRelease of your Play button
Code:

_root.monsterKillCount=0; // in case we placed any instances of the monster on this frame and 
the user has killed them
gotoAndPlay("start");


Background

Skip a few frames (enough so we can add credits, instuctions or whatever later if we wish. Skip 5
frames.).
Go to frame 6, label it "start" and draw a background on it. (boxes, hills, stars, etc.)
Select all objects in the background and add a transform motion: this allows it to remain for the rest of the level.
extend the transform mothion however long you want the level to go.

Now for the monsters!

Add a new layer called "Monster 1" and add a monster to it.
Draw a line using the freehand tool (you may want to set the smoothness to 0) or the curve tool.
This is the path for your monster to follow.
Now select both your monster and the path and insert a path motion.
Preview. Looks kinda weird, huh?
Exit the preview.
Add a new layer called "Stuff 1" and add whatever you want to be in front of this monster.
Make sure you draw this on the first frame of this level, and add transform motion to make it stick until
the end.

More Monsters!

Add a new layer for the next monster and rename it "Monster 2". Go to about the middle of the level and add another monster.
Select the monster and insert a transform motion.
On the first frame that this monster is in, place it all the way
on one side of the drawing area.
On the farthest frame that the monster is in, and move
it all the way to the other side of the drawing area.
Preview.
That... is really boring. And he moves really slow and is too easy to kill. This stinks. Ok, exit the preview. We are going to make the
monster hop around now.
Double click on the blue bar (for this monster) between the 2 blue dots
Wow! Another blue dot appeared!
Drag the monster to another position inside that
frame and resize or even rotate the monster (if you want).
Repeat this step to make the moster hop from one side of the screen to the other.
Preview.
Wow, that is pretty cool... But we need more monsters!! Exit the preview.

Make more layers, monsters (with motions), and layers containing stuff for monsters to hide behind whenever we feel like it.
This is starting to look pretty cool.

You Win (or Game Over)

Go to the end (the last frame of this level) and add the following ActionScript to onFrame:
Code:

if(_root.monsterKillCount==2){ //2 is the number of monsters in this level
   _root.monsterKillCount=0;
   gotoAndPlay("win");
} else {
   gotoAndPlay("lose");

Change 2 to the number of monsters you have.
Go to the next frame (after the end) and label it "win". Decorate it to let the user know they have won and include a button to go back to the menu frame (gotoAndStop("menu"), or to play again (gotoAndPlay("start")
Put "stop();" to this frame's onFrame.
Go to the next frame and do the same thing, but for "lose".


That's it, you're done!

That is correct, Jabapyth, to add more monsters, just drag and drop the monsters on and add transform motion (or path motion, if you like). (And update the monster count at the end of the level, of course.) In school, when you take any kind of higher level programming class, you are taught about encapusated code, meaning that adding another instance of something should be easy, with all the code contained within the "class" in this case, a "sprite". Also makes it easier to debug.
