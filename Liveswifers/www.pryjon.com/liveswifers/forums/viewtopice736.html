Today, in a conversation, I remembered a format that *should* be supported by the ajax animator, as it *will* be quite popular in the near future. the format is JavaFX. I think supporting all these formats will be a good thing. It may not only be the first web-based animation studio, but also the first JavaFX, Silverlight, or SMIL animation ide. One drawback, is that all these formats may confuse end-users. which will obviously be a bad thing. So, for the most part, i will try hiding the use of these formats, unless the user chooses to embed it into a site or something, and then pop up a friendly list of export options (with Flash selected as default and short summaries on why to pick each one).

Today, i finished the 'core' of the timeline. which isn't much. Right now, you see a quite clean (i'd say pretty - compared to the current v2 one) timeline. it does nothing yet, but the API is quite nice. It has been designed from the ground-up to support layers (the old one was just a bunch of hacks) nicely, and it should just work. Ideally, whatever button combinations you force the API into doing, things will still work fine (its not big/powerful enough to have bugs yet ).

I hope this project works. I want a working, stable version by the end of summer. I want people to use it - not at current stages. I'm not really advertising the Ajax Animator, and I don't really want to, and nobody really should. It shouldn't be advertised until 0.2 is ready. If you ask me, the current version is somthing I'd call an embarrassment. No, i'm not as embarassed about this 0.14.7 as much as the 0.13x releases (13 never was a really lucky number...) but the whole system sucked. All the releases are embarassments to web developers in general. They break one of the most fundamental rules: platform-independence. I lack support for IE. IE may suck, it may be a piece of crap, but why would I not support it.

I look back at my animator every once in a while. I find it horrifying. The color schemes are not really matching. Its quite ugly. It takes forever to load. Its crappily optimized. Half of the code hasn't even changed since the first release.

I'm sure I can't blame myself. I didn't have the tools then. I was still learning. And I still am. I won't be suprised if the project never reaches the 1.0 stage. Its not that special. Will there ever be a day when this project actually takes off, and i'm not afraid to talk about it. When there are no big bugs in the application. When people actually can use it. When documentation and tutorials are sufficient. When things just work.

Many if not most open-source projects are never blessed with a large number of contributers, this project is one of them.

I don't know exactly how to handle the project. I'm still a student, I have no idea.
