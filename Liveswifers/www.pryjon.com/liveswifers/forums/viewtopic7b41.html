Just a few suggestions I have:
*: Get hosted. Maybe think about applying for a SourceForge project so we can get a CVS/SVN repo?
*Decide on a programming language. Not sure if this has been done. My quick look around the forums didn't prove too fruitful so I may have missed something.
I reccomend Java for that (assuming there haven't been decisions).
Why? Too many reasons
1: it has a built in graphics and GUI toolkit (Swing and AWT). I can't say the same for most other languages
2: Its cross platform. The code should ALWAYS run the same. Sure, you might have to adapt for multiple OS's file hiarchy, but it does help.
3: Its currently one of the most popular (and if not the highest in demand) programming languages. We can find developers easily.
4: It has a HUGE variety of Flash design tools. I just came across this one: http://osflash.org/javaswf2
5: Its object oriented. We pretty much NEED this for what we're trying to do if we plan to draw objects on the design screen and not just simple shapes. I don't know how they managed to write the original liveswif in C(?) but it will be a lot easier with use of objects.
6: I can't think of one thing you can do in other languages that you can't do in Java. I've seen programs of every kind written in Java and develop for some myself (the arianne project, or one of my own). There is a wide community of Java developers and I find it a VERY easy language to learn (old C-type languages are hard. I can say that truthfully after learning both PHP and C)

And on a final note: Make it cross platform!!
PLEASE remember that there are other users that DON'T use Windows.
And currently, Mono is the only C# language for Linux and other OSes around and its still being developed. I'm not even sure it reached version 1.0 yet.
So, that being said, we can rule out these languages for multi-platform support:
C#/Mono
.NET

And we should get these too, as there are functions in these languages that are OS-specific:
C
C++
Other Windows-only or OS-specific languages

And this leaves:
Java (which I reccomend)
PHP (although GUIs would be difficult, we could go a step further and make it web based )

And why not C/C++? For one thing, C isn't object oriented. That would slow development and make the code barely readable. And not C++ because its practically C. C++ does not even prevent you from using old C-style code, thus leading to the problem I mentioned earlier. The only way to prevent people from writing the messy/crude code is to prevent it from being written in the first place.
