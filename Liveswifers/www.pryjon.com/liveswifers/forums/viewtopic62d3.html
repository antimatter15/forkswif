Xenos paradox state that no matter how much you divide something, it can never be zero (which he applied into motion)

I figured out how to make a sprite follow another sprite with delay
Code:

if (_root.followedsprite._x > _x){       
_x +=(_root.followedsprite._x-_x)/delay;
}
if (_root.followedsprite._x<_x){       
_x -=(_x-_root.followedsprite._x)/delay;   
}
if (_root.followedsprite._y > _y){       
_y +=(_root.followedsprite._y-_y)/delay;
}
if (_root.followedsprite._y<_y){       
_y -=(_y-_root.followedsprite._y)/delay;   
}


You can place this code onto the sprite that will be following another sprite.

You must replace "delay" with a number, the higher, the more delay
You can replace the instance name "followedsprite" with the instance name of the followed sprite

You can also use basic AI to make a sprite follow another sprite:

Code:

if(this._x > _root.followesprite._x){
this._x -= 3;
}
if(this._x < _root.followesprite._x){
this._x += 3;
}
if(this._y > _root.followesprite._y){
this._y -= 3;
}
if(this._y < _root.followesprite._y){
this._y += 3;
}


And if you want to use a variable for the speed replace the numbers after = with 'speed' and put this code in the onLoad event:

Code:

speed = 3;
