Well... it is hard to explain if you have never seen it... but first, on the top and left sides, it has the button toolbars and menus just like any other program; on the right, it has a properties box where you can set the color, size, etc. of whatever you are working on; in the middle it has the drawing area where you can click, drag, zoom, pan, and all that good stuff; but on the bottom... there is a command line area that you can type commands into like "line (point) (point)", "circle (origin) (radius)" "pline" (multiple lines stored in a single object, or poly line), "save", "open", basically all the functions are accessible from both the command line and the menus/buttons. AutoCadd also displays any errors or messages down in the command line window.

The reason I say this would be easier for development, is that it allows the programmer to not worry about the buttons so much, and focus on the functionality of the program. Then, when someone with less experience wants to make the buttons and toolbars, all they have do to is have the "click" function of the "circle button" (for example) routed to the "type circle in command line" function. For anyone reading who is not familiar with programming, just trust me that that part is really easy. So this way, the more experienced programmers can work on the hard stuff, and the beginning programmers can work on the easier stuff.

For any programmer curious about how it would look to be programming using Qt, http://www.trolltech.com/products/qt/learnmore/video/demos/browser is a nice screenshot movie demo.
