This is a tutorial about making a score counter. This was made to help moto.energy but anyone can use this tutorial.
Scenario:You want to show someone their score, say how many times a button has been clicked. How would you do that? This tutorial shows you how.
----------SECTION 1: THE OBJECTS----------
first, create a button. It doesn't matter how, just make it.
now we create a sprite. Frame 1 has the number 0, frame 2 has the number 1, frame 3 has the number 2, and so on. Frame 11 has 0 again (no, I'm not being redundant here, you see later)
----------SECTION 2: SETTING UP THE MOVIE----------
Drag three of the number sprites onto the main canvas. Select all three and go Tool->Align->Align to bottom.
Name the rightmost number sprite (the ones place) oneplace. Name the tens place (center number) tenplace. Finally, name the leftmost number sprite (hundreds place) hundredplace.
Drag a button onto the main canvas.
----------SECTION 3: THE CODE----------
In the onFrame event of the main canvas the stuff is on and the sprite's first frame, put this code:
Code:
stop();

Now for the important code. In the button's onRelease event, put in this code:
Code:
//ones place increment
_root.oneplace.nextFrame();
//now see if we add to the tens place
if (_root.oneplace._currentframe == 11) {
    _root.oneplace.gotoAndStop(1);
    _root.tenplace.nextFrame();
}
//do we add to the hundreds place?
if (_root.tenplace._currentframe == 11) {
    _root.tenplace.gotoAndStop(1);
    _root.hundredplace.nextFrame();
}
//if it is 999, goto next timeline frame
if(_root.oneplace._currentframe == 10 && _root.tenplace._currentframe == 10 && _root.hundredplace._currentframe == 10) {
    _root.nextFrame();
}

This is a breakdown of the code:

  • The first line of code increments the ones place
  • The first and second if blocks determine if we increment the tens and hundreds place, respectively.
  • The last if block see if the maximum limit has been reached

Add whatever in the next main timeline frame, and your done!
PM me if you need any help!
BONUS!
If you want the button to decrement, this code should work (I haven't gotten a chance to test it yet):
Code:
//ones place decrement
_root.oneplace.prevFrame();
//now see if we subtract the tens place
if (_root.oneplace._currentframe == 1) {
    _root.oneplace.gotoAndStop(10);
    _root.tenplace.prevFrame();
}
//do we subtract the hundreds place?
if (_root.tenplace._currentframe == 1 && _root.oneplace._currentframe ==1) {
    _root.tenplace.gotoAndStop(10);
    _root.oneplace.gotoAndStop(10);
    _root.hundredplace.prevFrame();
}
//if it is 0, goto next timeline frame
if(_root.oneplace._currentframe == 1 && _root.tenplace._currentframe == 1 && _root.hundredplace._currentframe == 1) {
    _root.nextFrame(); //make sure it is some sort of lose screen here.
}
