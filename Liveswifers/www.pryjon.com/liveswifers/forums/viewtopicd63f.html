Yes, you can do all those things, and more!!



Oh.. wait... you want me to tell you how to do it?
Oh bother, alright...
Well, for the enemy to die, you need to create a life variable like you said. On each frame, the enemy should check and see if it is hitting a bullet, and if it is, removeMovie() it and decrease its own life pointage by one.
Somewhat like the sample game in the default LiveSwif samples:
Code:
for (i in _root) {
   if (i.substring(0, 5) == "bullet") {
      if (this.hitTest(_root[i])) {
         unloadMovie(_root[i]);
         _root.lives -=1;
         _root.marklife.gotoAndStop(10-_root.lives);
         if (_root.lives <= 0) {
            _root.gotoAndStop(3);
         }
         
      }
   }   
}

Or something like that.

Firing rate is a little more complicated, but you just need to figure out under what conditions the gun should fire (while mouse is down? or only when it is clicked and released?) and maybe institute a timer if you want to have "automatic fire". (Remember, there is already a timer built in, the frame rate!! You can use that easily, if you wish!)

When is it you want it to fire?
When the mouse is released is what it already does (so you would just have to click it quicker to get it to fire more often).
If you want it to release a stream of, say, 5 bullets in rapid succession, just after the mouse button is released, you will need to make the onMouseUp be something like... set var bulletCount to 5, and the enterFrame to be something like, if(bulletCount), decrease bulletCount by one and fire a bullet.
If you want it to release bullets whenever the mouse is down, make the onMouseDown set var firing to true, the onMouseUp set firing to false, and the onEnterFrame to fire a bullet if(firing).

If you are using one of the ones with onEnterFrame, you can slow down the firing rate by setting a var holdFire to a number of frames that you want to wait, and in the onEnterFrame, if(holdFire), decrease the holdFire by one... otherwise (meaning that holdFire is 0), set the holdFire to the number you set it to in the first place and fire the bullet.


The code I gave before goes in the OnEnterFrame of the sprite getting shot at, and there should be a sprite called marklife in the _root to display the remaining life points.
