Well, first you have to check if the sprite is touching the other sprites bounding box.

Ill have five sprites, the main sprite (the player, character, main sprite etc.), the top wall, bottom wall, left wall, and right wall.

(The important to mention is that countering the movement of a main sprite to any obstacle is not simple. You will need the four compass sides, such as the up/down/left/right walls. Which leads me to another fact: you can not have and other obstacles other than rectangles and squares.)

So your obstacle should look like this:

[---Up Wall---]
= =
L R
e i
f g
t h
t
W.
W.
= =
[-Down Wall-]


Well, now that I've set it up...I forgot to check if your using a movement code that uses x/y +/-= speed;

If so carry on, if not, you're in the wrong place.

You see, when a sprite is moving right, it adds +speed to the _x variable, to the position changes every frame second. (Simulating the effects of movement). In order to stop it, we need to apply the same force, in the opposite direction. (Which is -speed to the _x variable).

Simply put:
The Up Wall code is
Code:
if(this.hitTest(_root.mainsprite){
_root.mainsprite._y -= speed;
}

The Down Wall code is
Code:
if(this.hitTest(_root.mainsprite){
_root.mainsprite._y += speed;
}

The Left Wall code is
Code:
if(this.hitTest(_root.mainsprite){
_root.mainsprite._x -= speed;
}

The Right Wall code is
Code:
if(this.hitTest(_root.mainsprite){
_root.mainsprite._x += speed;
}


As in all code posts, I strongly urge you to get the understanding of the code itself.
