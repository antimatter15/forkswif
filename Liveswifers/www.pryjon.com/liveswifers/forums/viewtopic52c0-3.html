We don't have any source code for the regular LiveSwif, otherwise we would use whatever language is was programmed in, unless we were really really ambitious and wanted to port it to Linux or something. But yes, we will be programming it totally from scratch. We do have the advantage of using libraries such as GTK for the buttons/text input areas/file open and save dialogs/etc., OpenGL for the rendering/drawing area, ming for the .swf output, and SciLexar for the ActionScript editing window. But we have to program all the code that ties that stuff together, and stuff like receives the input from the mouse for drop and drag/editing shapes, memory management of shapes, lines, etc., implentations of the buttons, file input and output (for the .lmv's or... I guess we will change them to .omv's for Openswif MoVies?), tons of work for reverse engineering of the .lmv files (if we try to be backward compatable), sound and font management, and plenty of other general stuff. That is also a good point about the .lmv files: document all your (important enough that you would want to edit it later type) work, because it may not be possible to import .lmv's into OpenSwif!
