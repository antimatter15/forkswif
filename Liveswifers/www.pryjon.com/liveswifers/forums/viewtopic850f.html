In light of recent events in this section, it has come to my attention there needs to be a clearer HitTest tutorial. Here we go.

The hitTest command checks to see if something bumped into something else. The ActionScript Dictionary, the best resource for any liveswifer, defines the hitTest syntax as the following:
Code:
myMovieClip.hitTest(x, y, shapeFlag); //usage 1
myMovieClip.hitTest(target); //usage 2

Usage 1

  • myMovieClip - the sprite the hitTest is being done on. Usually replaced by this.
  • x - the x coordinate of the hit area.
  • y - the y coordinate of the hit area.
  • shapeFlag - this is a weird one, it is set to two values.
    • True - means ONLY CHECK the bounding box of the sprite for a hit.
    • False - means to check the border of the sprite. We have never seen this value on the forums in use this way


Usage 2

  • myMovieClip - the sprite the hitTest is being done on. Usually replaced by this.
  • target - the second sprite that myMovieClip got bumped by.

Typical Usage
The only time you really see usage 1 on this board is when we want to see if the cursor bumped into us. The skeleton code is this:
Code:
if(this.hitTest(_root._xmouse, _root._ymouse, true)) {
//code here
}

Any other time, you use the other one like above, just replace the hitTest above with usage 2.
Examples
To make sure a sad face will react to cake on his mouth, this code will do it: (onEnterFrame)
Code:
if(this.hitTest(cake)) {
this.gotoAndStop("happy");
}


To make a mouse disappear when unless it is over a certain object, use this code: (onMouseMove)
Code:
if(this.hitTest(_root._xmouse, _root._ymouse, true)) {
mouse.show();
}
else {
mouse.hide();
}


That's the end of the tutorial! Hope you learned!
