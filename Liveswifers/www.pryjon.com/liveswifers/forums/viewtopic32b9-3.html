OK, here is how to do it...
Step 1, make a new sprite named "arrow" or "gun" or whatever (I am going to call it arrow since that is what I used.). Draw a pointer in it, and move the pointer so the pivot point around which is turns is at the very top left corner. Add an instance of it to the main stage ("Scene 1").
Step 2, add this code to the onload():
Code:
this.startX=this._x;
this.startY=this._y;
and add this code to the on mouse move:
Code:
x1=_root._xmouse;
y1=_root._ymouse;
this._rotation=(Math.atan((y1-startY)/(x1-startX))*180/Math.PI)+(x1<startX?180:0);

Step 3, Preview and enjoy!

Ok, first, let me say, that was the stupidest lunar eclipse I have ever seen which I saw just now... The moon did not even turn red! And then the sun came up and ruined it.

OK, on to business...
Create a 1px by 1px sprite of a dot and add it to the arrow, and call it pivot. (After this is finished, you may want to make it fully transparent by editing the sprite and removing the border from the shape or something.) Move it so that it is at the very top left corner by using the "Apply" toolbar (I mean, of course, the info bar). Make sure the instance name is "pivot".

Go back to the main stage and click the arrow. Remove the code from onload() that I gave you earlier, replace the code in onmousemove() with this:
Code:
x1=_root._xmouse;
y1=_root._ymouse;
x2=pivot._x+this._x;
y2=pivot._y+this._y;
this._rotation=(Math.atan((y1-y2)/(x1-x2))*180/Math.PI)+(x1<x2?180:0);

There you go! That should work... I think... You may need to add the _x and _y of the container the arrow is in if you have the arrow in another sprite. Kinda like:
Code:
x2=pivot._x+this._x+this._parent._x;
y2=pivot._y+this._y+this._parent._y;


Also, your math teacher will be impressed that you are using (y2-y1)/(x2-x1). And that you are using arcTan... but depending on how old you are, the teacher might figure out than you did not create that code.
